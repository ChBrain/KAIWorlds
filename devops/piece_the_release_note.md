# Piece: The Release Note
## The approval that opens the door.

## Owner
**Project:** DEVOPS

**Persona:** The Gatekeeper

## Place
[The Staging Environment](place_the_staging_environment.md).
Then with the [DevOps Engineer](position_devops_engineer.md).

## Load Bearing
Without it the chain cannot deploy.
With it the chain has permission
to place [The Change](piece_the_change.md) in the world.
Remove it and the deployment stops.
Keep it and the door opens.
What is on the other side
was never in the Staging Environment.

## Apparent
A document. A version number.
A list of changes.
A signature.
It looks like due diligence.
It is due diligence
for the object reaching production.
Not for the user receiving it.

## Yearbook
Signed in [Releasing](process_releasing.md).
The [Release Manager](position_release_manager.md) approved
what passed in [Testing](process_testing.md).
Staging resembled production.
The signature was obtained.
[The Change](piece_the_change.md) was approved
for a world it had never been in.

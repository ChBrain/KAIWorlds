# Process: Coding
## The logic that will become someone's experience.

## Owner
**Project:** DEVOPS

**Persona:** The Maker

## Initiates
[The Requirement](piece_the_requirement.md) exists.
The [Repository](place_the_repository.md) is ready.
[The Change](piece_the_change.md) runs here.
The Requirement stands in for [The User Space](place_the_user_space.md).
The user's context is not in the room.

## Direction
Toward [The Commit](piece_the_commit.md).
Not toward the user —
the Repository holds logic, not experience.

## Lever
The interpretation.
The [Software Developer](position_software_developer.md) converts
the Requirement into code.
What the Requirement meant
and what the code does
are not always the same thing.

## Echo
The Commit exists.
The [Build Engineer](position_build_engineer.md) receives it.
[Building](process_building.md) can begin.

# Piece: The Incident
## The system's name for what the user is experiencing.

## Owner
**Project:** DEVOPS

**Persona:** The Keeper

## Place
[The User Space](place_the_user_space.md).
Then [The Dashboard](place_the_dashboard.md).

## Load Bearing
Without it the chain has no mechanism
to hear that something is wrong.
With it the chain has a record
of what the system reported.
Not of what the user experienced.
Those are not always the same event.

## Apparent
A ticket. A severity level. An alert.
It looks like a problem named.
It is a system failure named.
The user's experience of that failure
is not in the ticket.

## Yearbook
Raised in [Operating](process_operating.md).
Sometimes.
Sometimes the system fails silently
and no Incident is raised
and the user absorbs the failure
without a word for it,
without a ticket,
without the chain knowing.
The Incident is the visible fraction.

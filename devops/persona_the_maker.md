# Persona: The Maker
## Title: Software Developer

## Owner
**Project:** DEVOPS

**Persona:** The Maker

They build what the requirement describes.
They decide what the requirement doesn't.

## Projection
Focused. Capable.
The editor is open.
The problem is interesting.
The user is not in the room.

## Action
Read the requirement.
Fill the gaps.
Ship the Commit.

## Shadow
Where the requirement was unclear
they made a decision.
The decision felt like interpretation.
It was also design.
The user will live with both
without knowing either happened.

## Tell
They close the ticket.
The ambiguity is inside the Commit now.

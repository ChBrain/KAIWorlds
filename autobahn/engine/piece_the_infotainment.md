# The Infotainment System

## Owner
- Project: Autobahn

## Place
In the dashboard. A screen. Connected.

## Load Bearing
Without it the modern car loses its interface layer. Navigation, music, phone, vehicle data, driver assist settings — all run through it. Remove it and the Driver must know the route, carry a separate device, manage everything that the system was managing silently. The system is load-bearing precisely because its presence is taken for granted.

## Apparent
A touchscreen, central. Larger in newer vehicles, smaller in older ones. The navigation running as a default — the route visible, the ETA updating. Apple CarPlay or Android Auto mirroring the Driver's phone. Streaming available. The vehicle's own data visible: tyre pressure, range, driver assist status. The volume controlled by touch or by voice. Updates downloaded overnight. The system knowing more about the journey than the Driver has consciously told it.

## Yearbook
In a saloon with a virtual cockpit: the route displayed at two scales simultaneously. The Driver glanced at it twice in two hours. It was correct both times.

In a sports saloon: the menu navigated from memory. Sport mode toggled once. The Driver did not toggle it back.

In a large executive car: the system responding to voice commands the Driver stopped using after the first month. The screen touched directly now. The system had anticipated a different interaction.

In a heavy truck: Predictive Powertrain Control integrated. The system shifting gears based on GPS gradient data. The Driver watching it work, deciding when to override. A relationship with automation that has no name.
